## What is Wundr?

### Primary Definition (Elevator Pitch)

Wundr is a comprehensive AI-powered development toolkit that supports developers through three
critical lifecycle stages: **Setup** (configuring developer machines), **Create** (scaffolding
production-ready projects), and **Govern** (maintaining code quality and standards). It's the
missing platform layer between your IDE and your codebase.

### Extended Definition (Product Page)

Wundr is a unified monorepo containing 14 published npm packages that work together to deliver an
integrated development experience. Built on Turborepo with TypeScript, Wundr combines:

- **Automated machine provisioning** for instant developer onboarding
- **Opinionated project templates** with best practices pre-configured
- **AI-powered code analysis** for continuous quality assurance
- **Governance automation** to prevent technical debt
- **Claude Flow integration** for agentic development workflows

## Tone & Voice Guidelines

### Brand Voice Attributes

#### 1. **Confident, Not Arrogant**

**Do**: "Wundr eliminates setup complexity so you can focus on building great products." **Don't**:
"Other tools are broken. Only Wundr works correctly."

#### 2. **Technical, Not Jargon-Heavy**

**Do**: "AST-powered code analysis detects duplicates and complexity hotspots." **Don't**:
"Leveraging synergistic paradigms to optimize code quality matrices."

#### 3. **Helpful, Not Condescending**

**Do**: "Getting started takes 5 minutes. Here's exactly what to do..." **Don't**: "Even a junior
developer can figure this out."

#### 4. **Opinionated, Not Dogmatic**

**Do**: "We chose Next.js because it provides the best DX for modern React." **Don't**: "Next.js is
the only acceptable framework. Everything else is wrong."

#### 5. **Developer-Centric, Not Marketing-Speak**

**Do**: "Parallel builds deliver 2.8x speedup through smart caching." **Don't**: "Revolutionary
next-generation paradigm-shifting technology."

### Writing Style Guidelines

#### Use Active Voice

**Do**: "Wundr automates machine setup in minutes." **Don't**: "Machine setup is automated by Wundr
in minutes."

#### Be Specific with Numbers

**Do**: "Onboard developers 90% faster (hours vs. days)" **Don't**: "Dramatically faster developer
onboarding"

#### Lead with Benefits, Follow with Features

**Do**: "Start coding immediately [benefit] with pre-configured development environments [feature]."
**Don't**: "Pre-configured environments [feature] let you start coding [benefit]."

#### Use "You" for Developers, "We" for Platform

**Do**: "You can scaffold a new project in 30 seconds." **Don't**: "The user interface allows
project creation."

#### Avoid Hedging Language

**Do**: "Wundr prevents technical debt through automated governance." **Don't**: "Wundr might help
reduce some potential technical debt issues."

### Formatting Conventions

#### Code Examples

- Always use syntax highlighting
- Include expected output when relevant
- Show realistic use cases, not toy examples
- Prefer complete working examples over snippets

#### Technical Documentation

- Start with "Quick Start" for time-pressed readers
- Provide both CLI and programmatic examples
- Include troubleshooting sections
- Link to related documentation

#### Marketing Content

- Use customer outcomes, not product features
- Include real metrics and benchmarks
- Show before/after comparisons
- Add social proof (testimonials, case studies)

## Glossary of Key Terms

### Core Concepts

**Wundr Platform** The complete ecosystem of 14 npm packages working together to deliver setup,
scaffolding, and governance automation.

**Three Features** The distinct capabilities Wundr provides: (1) Computer Setup, (2) Project
Creation, (3) Code Governance.

**Systematic Excellence** Wundr's philosophy of replacing ad-hoc development practices with
repeatable, automated quality systems.

**Governance Automation** Automated enforcement of code standards, patterns, and best practices
through continuous analysis and drift detection.

**AI-Native Development** Integration of AI capabilities directly into the development workflow, not
as an afterthought or add-on.

---

### Technical Terms

**AST (Abstract Syntax Tree)** A tree representation of code structure that enables deep analysis of
patterns, complexity, and dependencies.

**Drift Detection** Automated monitoring that identifies when code deviates from established
baselines or standards over time.

**Claude Flow** An orchestration system for AI agents that enables complex refactoring and
development workflows using Claude AI.

**SPARC Methodology** Specification â†’ Pseudocode â†’ Architecture â†’ Refinement â†’ Completionâ€”a
systematic approach to problem-solving.

**Governance Baseline** A snapshot of code quality metrics that serves as the benchmark for
measuring drift and regression.

**Turborepo** The high-performance build system Wundr uses for monorepo management, delivering 80%
cache hit rates.

**Developer Profile** A pre-configured set of tools and settings for specific roles (Frontend,
Backend, Full Stack, DevOps, etc.).

**Project Template** An opinionated project structure with all dependencies, tools, and
configurations pre-configured to Wundr standards.

**Computer Provisioning** The automated process of installing global development tools and
configuring a developer's machine.

**Wundr-Compliant Project** A project created with `wundr create` that includes all governance
baselines, AI integration, and best-practice configurations.

---

### Feature-Specific Terms

#### Computer Setup

- **Global Tools**: System-wide developer utilities (Git, Docker, Node.js, etc.)
- **Hardware-Adaptive Optimization**: Automatic tuning based on available system resources
- **Profile-Based Setup**: Role-specific tool and configuration installation

#### Project Creation

- **Opinionated Stack**: Pre-selected, integrated technology choices optimized for specific use
  cases
- **Scaffolding**: Automated generation of project structure, files, and configurations
- **Production-Ready Template**: Project setup that includes testing, CI/CD, and deployment configs

#### Code Governance

- **Quality Gate**: Automated checks that prevent merging code below quality thresholds
- **Complexity Metrics**: Measurements like cyclomatic complexity and cognitive complexity
- **Pattern Standardization**: Automated fixes to align code with approved patterns
- **Circular Dependency Detection**: Analysis that identifies problematic import cycles

## Brand Narrative

### The Wundr Story

**The Problem We Saw**

Modern development teams face a paradox: they have access to more tools than ever, yet spend less
time building and more time configuring, maintaining, and coordinating those tools. A new developer
joining a team loses their first week to setup. Starting a new project requires hours of decisions
about frameworks, linters, test runners, and CI pipelines. Existing codebases accumulate technical
debt despite everyone's best intentions.

The industry's response has been more toolsâ€”but point solutions only add to the chaos. What's
missing isn't another linter or another framework. It's a unified system that handles the entire
lifecycle systematically.

**Our Insight**

We realized that three critical moments determine software quality:

1. **Machine Setup** - If your environment is wrong from day one, everything built on it will be
   wrong
2. **Project Genesis** - The decisions made when starting a project echo through its entire lifetime
3. **Continuous Governance** - Quality doesn't happen once; it requires constant vigilance

Most organizations handle these moments manually, inconsistently, or not at all. The result: chaos
masquerading as agility.

**The Wundr Approach**

Wundr systematically addresses all three moments with opinionated automation:

**Computer Setup** eliminates the "works on my machine" problem by provisioning every developer's
environment identically, optimized for their hardware and role.

**Project Creation** embeds best practices from the first line of codeâ€”not as suggestions in a wiki,
but as enforced configurations in every template.

**Code Governance** prevents drift through continuous, automated analysisâ€”catching quality
regressions before they require painful refactors.

The result is a platform that makes excellence the path of least resistance.

**Why AI Changes Everything**

Traditional automation hit a ceiling: it could enforce rules but couldn't understand context. AI
changes the equation. With Claude Flow integration, Wundr doesn't just analyze codeâ€”it understands
intent. It doesn't just detect problemsâ€”it suggests fixes. It doesn't just scaffold templatesâ€”it
generates custom architectures.

This is AI-native development: where the machine handles the mechanical, freeing humans for the
creative.

**Where We're Going**

Wundr isn't finished. We're building toward a future where:

- Onboarding takes minutes, not days
- Every project starts production-ready
- Technical debt is caught before it happens
- AI and humans collaborate seamlessly
- Quality is automatic, not aspirational

**Join Us**

Wundr is open source because systematic excellence shouldn't be a competitive advantageâ€”it should be
table stakes. We're building the platform we wish existed when we started our careers. Whether
